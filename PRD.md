# PRD: 법률 챗봇 서비스

## 1. 제품 개요
법률 관련 질문에 대해 사용자가 입력한 내용에 기반하여 답변을 제공하는 웹서비스형 법률 챗봇을 개발한다. 본 서비스는 단순한 법률 정보 제공이 아니라, 사전예방을 목표로 하며 사용자가 법적 분쟁이나 문제를 조기에 인식하고 적절히 대응할 수 있도록 돕는 것을 핵심 가치로 한다.

## 2. 제품 목표
- 사용자가 법률 관련 질문을 입력하면 자연스럽게 답변을 받을 수 있도록 한다.
- 법률 지식의 접근성을 높여, 일반 사용자도 쉽게 법적 맥락을 이해할 수 있도록 한다.
- 분쟁 발생 전 사전 예방에 도움이 되는 안내를 제공한다.
- 법률 자문을 대체하는 것이 아니라, 참고 및 예방형 보조 서비스로 활용되도록 한다.

## 3. 핵심 기능
### 3.1 사용자 질문 응답
- 사용자가 질문을 입력하면 챗봇이 답변을 제공한다.
- 질문은 법률, 계약, 분쟁, 채무, 소송 등 민사법 관련 주제를 포함할 수 있다.

### 3.2 예방형 안내
- 답변과 함께 다음과 같은 정보를 함께 제공한다.
  - 주의할 점
  - 추가로 확인해야 할 사항
  - 사전에 준비해야 할 행동
  - 필요 시 전문가 상담 권유

### 3.3 웹서비스 인터페이스
- 사용자는 웹 브라우저를 통해 질문을 입력하고 챗봇 응답을 확인한다.
- 채팅형 UI 형태로 구현한다.

## 4. 모델 및 데이터
### 4.1 모델
- 모델명: EXAONE/EXAONE-4.0-1.2B
- 법률 도메인 답변 생성에 활용한다.

### 4.2 학습 데이터
- AI Hub에서 다운로드한 민사법 데이터를 사용한다.
- 데이터는 법률 문서, 민사법 관련 내용, 예방형 안내와 연결될 수 있는 지식 기반으로 정제하여 활용한다.

## 5. 아키텍처 방향
본 서비스는 초기 구현 속도와 실용성을 고려하여 RAG 기반 구조를 채택한다.

### 5.1 아키텍처 개요
- 사용자의 질문을 받으면 먼저 관련 법률 문서를 검색한다.
- 검색된 내용을 바탕으로 모델이 답변을 생성한다.
- 답변에 법률 근거, 주의사항, 예방 조치, 전문가 상담 권유를 포함한다.

### 5.2 기술 스택
- 백엔드 언어: Python
- 프론트엔드 언어: JavaScript
- 웹 서버: FastAPI
- 프론트엔드 프레임워크: React
- LLM: EXAONE/EXAONE-4.0-1.2B
- 임베딩 모델: BAAI/bge-m3
- 벡터 데이터베이스: FAISS
- 모델 실행: Hugging Face Transformers
- 검색 방식: 임베딩 기반 유사도 검색, 키워드 검색 보조 활용
- 데이터 처리: Python 기반 전처리 및 문서 청크 분할

## 6. 시스템 구성 요소
- 프론트엔드: 질문 입력과 답변 표시를 담당한다.
- 백엔드: 사용자의 요청을 처리하고, 검색 모듈 및 모델 모듈과 연동한다.
- 검색 모듈: 관련 법률 문서와 조항을 조회한다.
- 모델 모듈: 검색 결과를 바탕으로 답변을 생성한다.
- 응답 포맷팅 모듈: 답변을 핵심 답변, 근거, 주의사항, 예방 조치, 상담 권유 형식으로 정리한다.
- 안전성 모듈: 개인정보 경고, 법률 자문 한계 안내, 고위험 상황 상담 권유를 담당한다.

## 7. 서비스 동작 흐름
1. 사용자가 웹 화면에서 질문을 입력한다.
2. 프론트엔드가 백엔드 API로 요청을 보낸다.
3. 백엔드가 검색 모듈을 실행해 관련 법률 내용을 찾는다.
4. 검색 결과를 모델에 전달한다.
5. 모델이 답변을 생성한다.
6. 백엔드가 응답을 정리해 프론트엔드로 반환한다.
7. 사용자는 답변과 예방 안내를 확인한다.

## 8. 서비스의 한계 및 안전성
- 본 서비스는 법률 자문 서비스가 아니며, 실제 법률 판단을 대체하지 않는다.
- 민감한 개인정보나 사건별 사실관계가 포함된 정보는 입력하지 않도록 안내한다.
- 고위험 또는 중대한 분쟁 상황에서는 전문가 상담을 권유하는 방식으로 설계한다.

## 9. 기대 효과
- 일반 사용자가 법률 관련 문제를 조기에 인식할 수 있다.
- 사전 예방 관점에서 문제 대응 능력을 높일 수 있다.
- 법률 정보 접근성을 향상시킬 수 있다.

## 10. 개발 방향
- 초기에는 웹서비스 형태로 빠르게 구현한다.
- 질문 응답 기능과 예방형 안내 기능을 우선 구현한다.
- 이후 데이터 품질 개선, 모델 성능 향상, 응답 정확도 향상을 단계적으로 진행한다.
