# PRD: 대한민국 생활 법률 AI 챗봇 웹 서비스

## 1. 프로젝트 개요
대한민국 생활 법률 AI 챗봇은 사용자가 법률 관련 질문을 쉽게 입력할 수 있도록 설계된 웹 기반 AI 서비스이다. 사용자는 텍스트 입력뿐 아니라 마이크를 통해 우리말로 직접 말해 질문할 수 있으며, Web Speech API를 통해 음성을 텍스트로 자동 변환한 뒤 AI 답변을 받을 수 있다.

본 프로젝트는 초기 MVP 단계에서 웹 서비스 형태로 구현하며, 프론트엔드와 백엔드를 분리하여 개발하고 배포한다. 프론트엔드는 Vercel에 배포하고, 백엔드는 별도의 호스팅 환경에 배포한다.

---

## 2. 제품 목표
- 사용자가 웹 UI를 통해 텍스트로 법률 관련 질문을 입력할 수 있도록 한다.
- 사용자가 마이크를 통해 우리말로 말하면 Web Speech API가 음성을 텍스트로 변환해 질문으로 전달할 수 있도록 한다.
- 백엔드에서 RAG 기반 검색과 Gemini 응답 생성을 수행한다.
- 프론트엔드는 채팅형 인터페이스로 답변과 근거를 사용자에게 보여준다.
- MVP 단계에서는 로그인, 회원관리, 결제, 관리자 기능은 제외한다.

---

## 3. 문제 정의
현재 많은 법률 정보 서비스는 텍스트 입력 중심이며, 사용자가 직접 말로 질문하는 경험을 제공하지 않는다. 특히 법률 정보는 문장으로 질문하는 경우가 많아, 사용자가 손쉽게 질문할 수 있는 인터페이스가 필요하다.

본 서비스는 음성 입력 기능을 통해 사용자 경험을 개선하고, 법률 정보 탐색을 더 직관적이고 접근성 있게 만든다.

---

## 4. 솔루션 개요
사용자는 웹 페이지에서 다음 방식으로 질문할 수 있다.

1. 텍스트 입력
2. 마이크를 통한 음성 입력
3. 음성 입력 시 Web Speech API가 자동으로 텍스트 변환
4. 변환된 질문이 백엔드로 전달됨
5. 백엔드가 RAG 검색 및 Gemini 응답 생성
6. 프론트엔드가 결과를 화면에 표시

---

## 5. 핵심 기능 요구사항

### 5.1 프론트엔드 기능
- 메인 채팅 화면 제공
- 질문 입력창 제공
- 전송 버튼 제공
- 로딩 상태 표시
- 마이크 버튼 제공
- 음성 인식 시작/중지 기능 제공
- 음성 인식 결과를 입력창에 자동 반영
- 답변 카드 형태로 결과 표시
- 근거 문서(법령/판결문) 목록 표시
- 에러 메시지 표시
- 반응형 레이아웃 지원

### 5.2 백엔드 기능
- 질문 수신 API 제공
- RAG 검색 로직 연동
- Gemini API 호출 및 응답 생성
- 답변과 근거 문서를 구조화된 형태로 반환
- 헬스체크 엔드포인트 제공
- 예외 처리 및 로깅 지원

### 5.3 AI 엔진 기능
- 법률/판결문 데이터 기반 RAG 검색 수행
- 검색 결과를 근거로 답변 생성
- 답변에서 관련 법령 및 판결문 정보 표시
- 추측 금지, 쉬운 한국어 사용, 일반 정보 안내 포함
- 구조화된 응답 형식 사용

---

## 6. 차별화 포인트
- 기존 텍스트 중심 챗봇과 달리, 사용자가 한국어로 직접 말해도 시스템이 자연스럽게 질문으로 변환한다.
- 음성 입력 기능은 접근성을 높이고, 법률 정보 탐색 경험을 더 직관적으로 만든다.
- 브라우저 기반 Web Speech API를 활용하여 초기 MVP에서 빠르게 구현 가능하다.

---

## 7. 사용자 시나리오
1. 사용자가 웹사이트에서 텍스트로 질문을 입력하거나 마이크로 음성 입력을 시작한다.
2. Web Speech API가 음성을 텍스트로 변환한다.
3. 프론트엔드가 변환된 텍스트를 백엔드 API로 전송한다.
4. 백엔드가 RAG 검색 후 Gemini 응답을 생성한다.
5. 프론트엔드가 답변과 근거를 화면에 표시한다.
6. 사용자가 추가 질문을 이어서 입력한다.

---

## 8. 비기능 요구사항
- 사용자는 웹 브라우저에서 쉽게 접근할 수 있어야 한다.
- 응답은 가능한 한 빠르게 표시되어야 한다.
- 프론트엔드와 백엔드는 분리 배포되어야 한다.
- 환경 변수 기반으로 민감 정보 관리가 가능해야 한다.
- CORS, HTTPS, 보안 설정이 적절히 구성되어야 한다.
- 로그와 예외 처리가 명확해야 한다.

---

## 10. 데이터 구성 방향
- 법률 데이터는 JSON 형태의 판결문 및 법령 문서를 사용한다.
- 데이터는 사전 전처리 과정을 거쳐 텍스트 청크로 분할한다.
- 각 청크는 메타데이터를 유지한다.
- 검색 시 관련 법령/판결문을 Top-K로 반환한다.

---

## 11. MVP 범위
다음 기능을 MVP로 정의한다.
- 텍스트 기반 질문/답변 기능
- Web Speech API 기반 음성 입력 기능
- 음성 인식 결과 자동 입력 및 전송 기능
- 답변과 근거 문서 표시 기능
- 기본 에러 처리 및 로딩 상태 UI
- 백엔드/프론트엔드 분리 배포 구조

---

## 12. 제외 범위
다음 기능은 MVP 범위에 포함하지 않는다.
- 로그인 및 회원가입
- 사용자별 계정 관리
- 결제 및 구독 기능
- 관리자 대시보드
- 대화 기록 저장 및 관리
- 고급 법률 상담 기능
- 다국어 지원

## 14. 검증 기준
- 텍스트 입력과 음성 입력이 모두 정상 동작하는지 확인한다.
- Web Speech API가 음성을 텍스트로 변환하는지 확인한다.
- 질문 입력 후 답변 생성 흐름이 정상 동작하는지 확인한다.
- 백엔드 헬스체크가 정상인지 확인한다.
- 프론트엔드에서 API 응답이 정상 표시되는지 확인한다.
- 배포 환경에서 CORS 및 환경 변수 연결이 정상인지 확인한다.

---

## 15. 성공 지표
- 사용자가 텍스트 또는 음성으로 질문을 입력해 답변을 받는 데 성공한다.
- 음성 입력 기능이 서비스의 주요 차별점으로 인식될 수 있다.
- 프론트엔드와 백엔드가 분리된 구조로 안정적으로 운영된다.

---

## 16. 결정 사항
- 웹 서비스 형태로 구현한다.
- 프론트엔드는 Vercel에 배포한다.
- 백엔드는 별도 배포 환경에 분리한다.
- 초기 MVP는 로그인 없는 단일 사용자형 챗봇으로 설정한다.
- Web Speech API를 활용한 음성 입력 기능을 핵심 차별점으로 포함한다.
